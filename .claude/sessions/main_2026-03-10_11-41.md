# Session: main — 2026-03-10_11-41

## What we worked on
This session continued from the next-intl migration (previous session). We debugged and fixed two critical bugs (language switching not working, layout shifting on mobile), then added a new `/support` page with bilingual FAQ accordion.

## Key decisions
- **Middleware location**: Moved `middleware.ts` from project root to `src/middleware.ts` — Next.js silently ignores middleware at root when using `src/` directory structure.
- **Custom middleware instead of next-intl's `createMiddleware`**: The `localePrefix: "never"` + `[locale]` folder approach caused 404s. Reverted to flat `src/app/` structure with a custom middleware that sets both a response cookie and an `x-locale` request header.
- **Request header for locale passing**: Cookies set in middleware response aren't visible to `cookies()` in server components on the same request. Added `x-locale` request header to ensure first-visit locale detection works immediately.
- **Font CSS variable fix**: Replaced all 26 instances of `font-family:'Cormorant Garamond',serif` with `font-family:var(--font-cormorant),serif` to use next/font's metric-adjusted fallback and eliminate mobile layout shift.
- **Support page structure**: Based on research of indie app support pages (Bear, Things, Day One, ABY Journal), chose a minimal structure: lead text, 8-question FAQ accordion, contact email, helpful links. Reuses `.prose-page` styling from privacy page.
- **Footer update**: Replaced "About" link (pointed to `#`) with "Support" link pointing to `/support`.
