# Session: main — 2026-03-11_13-50

## What we worked on
Optimized the Melior landing page layout for desktop (PC) screens. The mobile layout was already good; this session focused on making all sections look polished on wider viewports (1024px+).

## Key decisions
- **Hero open book**: Desktop shows an open book (two pages spread flat) instead of the mobile closed book. Both pages use the same warm paper color (`#F0EBE3`→`#E6E0D6`) for cohesion. Kept centered vertical layout (rejected horizontal book-left/text-right layout as it felt disjointed).
- **No scale tricks for YOY**: Rejected progressive card scaling for year-over-year section — looked mechanical. Instead used opacity-based narrative (past faded → present vivid → future dim) with a timeline line. Removed dots/markers after alignment issues.
- **Showcase 3-phone fan**: Restored the 3-phone fan arrangement for desktop using existing `.phones`/`.ph` CSS classes. Fixed a `clamp()` bug where negative values had min/max swapped, causing asymmetry.
- **Promise 2x2 grid**: Changed from 4-column to 2x2 grid on desktop, vertical centered layout per card (icon → label → desc stacked).
- **Chinese font weight**: Reduced `.toc-name` from `font-weight:500` to `400` in Chinese mode — PingFang SC medium weight looked too heavy for chapter names.

## Useful context
- Dev server: `npm run dev` (currently running on port 3001, port 3000 was occupied)
- The site at localhost:3000 may show a different/older version — check which port the dev server is on
- Browser cache was aggressive during this session — always hard-refresh (`Cmd+Shift+R`) when checking CSS changes
- `clamp(min, val, max)` requires min ≤ max — negative values are a common trap (e.g. `clamp(-180px, -14vw, -100px)` not `clamp(-100px, -14vw, -180px)`)
- `.book-open` is `display:none` by default, shown only at `min-width:1024px`; `.book-closed` is the inverse
- YOY quote text uses `<br/>` in i18n JSON for mobile line breaks — hidden on desktop via `.yoy-q br{display:none}`
