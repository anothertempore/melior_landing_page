# Session: main — 2026-03-09_13-54

## What we worked on
Migrated the Melior landing page from a custom `useState`-based i18n system to `next-intl`, with server-side language detection via middleware to eliminate flash of wrong language. Also fixed a mobile layout shifting issue caused by font loading.

## Key decisions
- **next-intl without `[locale]` URL segments**: Tried the `localePrefix: "never"` + `[locale]` folder approach but it didn't work — next-intl middleware didn't rewrite URLs for `localePrefix: "never"`. Reverted to flat `src/app/` structure with a custom middleware instead of `createMiddleware`.
- **Custom middleware over next-intl middleware**: Wrote a plain Next.js middleware that reads `Accept-Language` on first visit and sets a `NEXT_LOCALE` cookie + `x-locale` request header. The header ensures `request.ts` gets the correct locale on the same request (cookies set in response aren't visible to server components on that request).
- **English-first site**: `defaultLocale: "en"`, metadata in English, `<html lang="en">` as default. `:lang(zh-Hans)` CSS overrides for Chinese font (PingFang SC) on top of English base styles.
- **Font variable for CLS fix**: Replaced all 26 instances of `font-family:'Cormorant Garamond',serif` with `font-family:var(--font-cormorant),serif` to use `next/font`'s auto-generated metric-adjusted fallback font, eliminating layout shift during font swap on mobile.

## Useful context
- **Middleware location gotcha**: With `src/` directory, Next.js expects `src/middleware.ts`, NOT `middleware.ts` at project root. This caused the middleware to silently not run.
- **Cookie timing gotcha**: Cookies set by middleware in the response are NOT visible to `cookies()` in server components on the same request. Must use request headers (`x-locale`) to pass data from middleware to server components within the same request cycle.
- **next-intl `localePrefix: "never"` gotcha**: Despite the name, the middleware still tries to do internal URL rewrites. Without a `[locale]` folder segment, pages 404. The workaround was to skip `createMiddleware` entirely and write a custom one.
- **Font CLS gotcha**: `next/font` generates a CSS variable with metric-adjusted fallback fonts, but only if you USE that variable. Directly referencing `'Cormorant Garamond',serif` bypasses the fallback and causes layout shift.
- Reference project: `/Users/kexin/Documents/2026/ai-history` uses next-intl with `[locale]` URL segments — different approach.
