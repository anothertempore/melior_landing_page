# Session: main — 2026-03-09_13-07

## What we worked on
Modified the language detection logic in `src/lib/i18n.ts` so the app detects the user's browser/system language on first visit instead of always defaulting to Chinese. Return visitors still get their manually chosen language from localStorage.

## Key decisions
- **Browser detection over SSR detection**: Decided against adding Next.js middleware for server-side `Accept-Language` detection. The current client-side `navigator.language` approach in `useEffect` is sufficient, even though it may cause a brief flash of the SSR default language.
- **SSR defaults to English**: Changed the SSR fallback from `"zh"` to `"en"` per user request.
- **Detection logic order**: localStorage (saved preference) > `navigator.language` (browser/system) > fallback to `"en"`.

## Changes made
- `src/lib/i18n.ts` — Modified `getSavedLang()`:
  - SSR fallback changed from `"zh"` to `"en"`
  - Added `navigator.language` detection when no localStorage value exists
  - If browser language starts with `"zh"`, returns `"zh"`; otherwise returns `"en"`

## Open questions
